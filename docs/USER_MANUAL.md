# 实验课签到与成绩管理助手使用说明书

本文档面向实际使用者，介绍如何用本软件完成一次实验课项目的建立、名单导入、签到表导出、作业收取、自动评阅和成绩导出。

## 1. 软件定位

实验课签到与成绩管理助手是一款本地桌面软件，适合需要处理以下工作的课程：

- 从 Excel 导入学生名单。
- 从 Excel 导入选课时间或实验分组。
- 生成实验课签到表。
- 从邮箱或本地文件夹收集学生提交的报告。
- 批量匹配学生和作业文件。
- 自动给出初步成绩，并允许人工复核。
- 导出最终成绩单。

软件默认在本机处理数据，不需要把学生信息上传到外部服务器。

## 2. 启动软件

开发环境下启动：

```bash
npm install
npm run dev
```

启动后会打开桌面应用窗口。开发模式下终端可能会显示一些 Electron 或 DevTools 信息，只要应用窗口能正常使用，一般不影响操作。

## 3. 项目管理

### 3.1 新建项目

进入软件后，可以在顶部点击“新建”，创建一个新的课程项目。

建议填写清楚：

- 课程名称
- 学期
- 教师或管理者

项目保存后，软件会在本地建立独立项目目录，用于保存名单、导入文件、作业附件、评分结果和导出文件。

### 3.2 打开项目

点击“打开”可以选择之前保存的项目文件。打开后，学生名单、时间分组、作业匹配和成绩都会恢复到上次保存的状态。

### 3.3 保存项目

点击“保存”会把当前项目状态写入本地项目文件。建议在完成名单导入、邮箱导入或批量评分后及时保存。

## 4. 导入学生名单

进入“学生名单”页面，选择学生名单 Excel 文件。

软件会尝试自动识别常见字段，例如：

- 学号
- 姓名
- 年级
- 班级
- 院系

导入后请检查表格内容是否正确，尤其是学号和姓名。后续签到表、邮箱附件匹配和成绩导出都会依赖这两个字段。

## 5. 导入时间分组

进入“时间分组”页面，选择选课时间或实验时间 Excel 文件。

软件会读取时间段信息，并尝试将学生匹配到对应实验时间。导入后可以检查：

- 时间段名称是否正确。
- 每个时间段的人数是否合理。
- 是否有学生未匹配。
- 是否需要手动设置组名。

设置完成后保存项目。

## 6. 生成签到表

进入“签到表生成”页面。

常用流程：

1. 检查课程、学期和时间分组。
2. 调整签到表布局。
3. 预览表格内容。
4. 导出 Excel 签到表。

导出的签到表可以直接打印，也可以在 Excel 中继续微调。

## 7. 收集和匹配作业

进入“报告批改”页面。软件支持两种主要方式收集作业。

### 7.1 从本地文件夹扫描

如果已经把学生作业下载到本地，可以点击“选择报告文件夹”，然后点击“扫描并匹配”。

软件会根据文件名中的学号或姓名尝试匹配学生。

### 7.2 从邮箱导入

点击“邮箱导入”，填写 IMAP 邮箱配置。

常见字段：

- 服务器：邮箱 IMAP 服务器地址。
- 端口：SSL/TLS 通常使用 `993`。
- SSL/TLS：一般建议开启。
- 用户名：邮箱账号。
- 密码：邮箱密码或客户端授权码。
- 邮箱目录：通常是 `INBOX`。
- 日期范围：只扫描指定时间范围内的邮件。
- 标题关键词：可选，用来缩小扫描范围。
- 最大数量：限制本次最多处理多少封邮件。

匹配优先级：

1. 发件人显示名匹配学生姓名。
2. 附件名包含学生学号。
3. 附件名包含学生姓名。
4. 邮件标题包含学生学号。
5. 邮件标题包含学生姓名。

未匹配到学生的附件不会保存到作业目录。已经导入过的附件会自动去重，重复扫描不会重复下载。

## 8. 批改与自动评阅

### 8.1 打开学生文件

在“报告批改”页面点击学生的报告文件，软件会使用系统默认程序打开该文件。

例如：

- docx 使用 Word 或 WPS 打开。
- pdf 使用默认 PDF 阅读器打开。

### 8.2 自动评阅

点击“自动评阅”后，软件会批量读取已匹配的报告文件，并给出初步分数。

当前自动评阅主要参考：

- 报告总有效内容量。
- 思考题、问答、解释和总结类内容量。
- 全班提交情况中的相对位置。

自动评阅结果会直接填入成绩栏，并保留可展开的评价说明。教师可以根据实际情况手动修改分数。

注意：自动评阅适合减少重复性初评工作，不建议完全替代人工复核。扫描版 PDF、图片型报告或异常格式文件可能无法完整读取。

## 9. 导出成绩单

进入“成绩导出”页面。

导出前建议检查：

- 是否所有学生都有成绩。
- 是否有学生缺交。
- 是否有报告文件未匹配。
- 是否需要人工调整自动评分结果。

确认后点击导出，软件会生成 Excel 成绩单。

## 10. 数据和隐私

项目数据默认保存在本地项目目录下。典型结构如下：

```text
.lab-attendance-assistant/
  projects/
    课程名 - 学期/
      project.json
      imports/
      submissions/
        email/
      exports/
```

其中可能包含：

- 学生名单。
- 学生成绩。
- 邮箱导入配置。
- 学生提交的报告附件。
- 导出的签到表和成绩单。

如果保存了邮箱密码，当前版本会把它以明文形式保存在本地项目文件中。请不要把本地项目数据目录上传到公开仓库或发送给无关人员。

## 11. 常见问题

### 11.1 为什么邮箱登录失败？

常见原因包括：

- 用户名格式不正确，需要使用完整邮箱地址。
- 密码不是客户端授权码。
- 邮箱没有开启 IMAP。
- 学校邮箱限制了第三方客户端登录。
- 服务器、端口或 SSL/TLS 设置不正确。

### 11.2 为什么有些 docx 没读出来？

可能原因包括：

- 文件不是标准 docx。
- 文件损坏。
- 内容主要是图片或扫描件。
- 文档结构异常，文本提取库无法完整解析。

这类文件可以手动打开检查，并人工输入分数。

### 11.3 为什么自动评分不等于人工评分？

自动评分只根据可读取文本和相对分布进行估计，无法完全理解实验结果是否正确。它适合给出初稿分数，最终成绩仍应以教师判断为准。

### 11.4 为什么 GitHub 仓库里没有项目数据？

项目数据包含隐私信息，默认被 `.gitignore` 排除。仓库只保存软件源码和文档。

## 12. 推荐工作流

一次完整课程处理流程建议如下：

1. 新建项目。
2. 导入学生名单。
3. 导入时间分组。
4. 检查学生和时间匹配结果。
5. 导出签到表。
6. 从邮箱或本地文件夹导入作业。
7. 扫描并匹配学生报告。
8. 自动评阅。
9. 人工复核关键样本。
10. 导出成绩单。
11. 保存项目。
